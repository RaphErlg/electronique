🧱 Les briques à connaître par cœur

Presque tous les exercices se construisent avec ces quelques blocs. Si tu les maîtrises, tu peux reconstruire n'importe quel programme.

Le squelette + les commandes E/S 
void setup() {        // exécuté UNE SEULE FOIS au démarrage
  pinMode(8, OUTPUT);        // 8 en sortie (LED)
  pinMode(2, INPUT_PULLUP);  // 2 en entrée avec résistance interne
  Serial.begin(9600);        // ouvre la liaison série (moniteur)
}

void loop() {         // répété À L'INFINI
  int e = digitalRead(2);    // lit une entrée → HIGH ou LOW
  digitalWrite(8, HIGH);     // met une sortie à 5 V (LOW = 0 V)
}
CommandeRôle

digitalWrite(p, HIGH/LOW)allume / éteint une sortie
digitalRead(p)lit une entrée logique

analogWrite(p, 0..255)sortie PWM (broches ~)

⭐ INPUT_PULLUP : « appuyé = LOW »

Avec INPUT_PULLUP, l'Arduino branche une résistance interne vers le 5 V. La broche est donc à HIGH au repos. Le bouton, lui, relie la broche à la masse (GND).

👉 Bouton relâché = HIGH, bouton appuyé = LOW. C'est l'inverse de l'intuition ! D'où le test :

if (digitalRead(BP) == LOW) {   // ← appuyé
  // ...
}

Avantage : pas de résistance externe à câbler pour le bouton (juste bouton entre la broche et GND).

⭐ Le clignotement non bloquant (le patron millis)

C'est LE réflexe à avoir : delay() fige tout le programme. Avec millis(), on « regarde l'heure » sans bloquer, ce qui permet de faire plusieurs choses à la fois (clignoter ET lire un bouton).

unsigned long tPrecedent = 0;
const unsigned long INTERVALLE = 500;   // ms
bool etat = LOW;

void loop() {
  if (millis() - tPrecedent >= INTERVALLE) {  // assez de temps passé ?
    tPrecedent = millis();   // on note l'instant du basculement
    etat = !etat;            // on inverse
    digitalWrite(LED, etat);
  }
  // la loop reste libre pour le reste
}

À retenir : if (millis() - tPrecedent >= INTERVALLE) { tPrecedent = millis(); ... }

⭐ Niveau vs front, et l'anti-rebond

Niveau = « est-ce appuyé maintenant ? » (LED qui suit le bouton). Front = « vient-on de passer de relâché à appuyé ? » (un seul événement par appui — pour un télérupteur, un compteur…).

Détecter un front = comparer le niveau actuel au précédent :

bool dernier = HIGH;
void loop() {
  bool niveau = digitalRead(BP);
  if (niveau == LOW && dernier == HIGH) {   // front d'appui (HIGH→LOW)
    // ... action UNE fois
  }
  dernier = niveau;
}

Anti-rebond (debounce) : un bouton mécanique « rebondit » quelques ms → un appui peut être lu comme plusieurs. On ne prend le changement en compte que s'il est stable depuis ~30 ms (voir le télérupteur, exercice 4).

1 · Faire clignoter une LED (millis)

Manip 3
Cahier des chargesFaire clignoter une LED avec une période raisonnable, sans delay() (méthode non bloquante).
🔌 Le câblage

R = (5 − 2) / 0,02 = 150 Ω minimum ; 220 Ω est une valeur sûre courante.

🧠 La logique & les commandes

Une seule sortie. On applique le patron millis() : toutes les 500 ms, on inverse l'état de la LED. Commandes : pinMode, millis(), digitalWrite.

💻 Le code commenté 
#define LED 9
unsigned long tPrecedent = 0;
const unsigned long INTERVALLE = 500;   // 500 ms allumé, 500 éteint → 1 Hz
bool etatLED = LOW;

void setup() {
  pinMode(LED, OUTPUT);
}

void loop() {
  if (millis() - tPrecedent >= INTERVALLE) {
    tPrecedent = millis();   // mémorise l'instant du basculement
    etatLED = !etatLED;      // inverse l'état
    digitalWrite(LED, etatLED);
  }
}
✅ À tester / pièges
  • La LED clignote à 1 Hz (1 s allumée, 1 s éteinte... non : 0,5 s chacune → cycle de 1 s).
  • Mets INTERVALLE à 1000 pour ralentir et bien voir.
  • Piège : oublier tPrecedent = millis(); → la LED reste figée.

2 · Clignoter + une LED qui suit un bouton

Manip 4
Cahier des chargesUne LED clignote (période longue), et une autre LED s'allume selon un bouton poussoir. Sans delay().
🔌 Le câblage
🧠 La logique & les commandes

Deux choses « en même temps » → c'est l'intérêt de millis(). Bloc 1 : le patron millis fait clignoter D8. Bloc 2 : on lit le bouton et on recopie son état (niveau) sur D10. delay() serait impossible ici (il bloquerait le clignotement).

💻 Le code commenté 
#define LED_CLIGNOTE 8
#define LED_BOUTON   10
#define BP           2
unsigned long tPrecedent = 0;
const unsigned long INTERVALLE = 500;
bool etatClignote = LOW;

void setup() {
  pinMode(LED_CLIGNOTE, OUTPUT);
  pinMode(LED_BOUTON, OUTPUT);
  pinMode(BP, INPUT_PULLUP);
}

void loop() {
  // 1) clignotement non bloquant
  if (millis() - tPrecedent >= INTERVALLE) {
    tPrecedent = millis();
    etatClignote = !etatClignote;
    digitalWrite(LED_CLIGNOTE, etatClignote);
  }
  // 2) la 2e LED suit le bouton (appuyé = LOW)
  bool appuye = (digitalRead(BP) == LOW);
  digitalWrite(LED_BOUTON, appuye);
}
✅ À tester / pièges
  • La 1ère LED clignote sans s'arrêter quand tu presses le bouton → preuve que c'est bien non bloquant.
  • Piège : si tu avais utilisé delay(500), le bouton ne répondrait qu'une fois par seconde.

3 · Deux LED à des fréquences différentes

Manip 5
Cahier des chargesLED 1 clignote à 1 Hz, LED 2 à 0,8 Hz (duty cycle 50 % chacune).
🔌 Le câblage

2 LED indépendantes : D8 → R 220 Ω → LED1 → GND et D10 → R 220 Ω → LED2 → GND (même principe que l'exercice 1, deux fois).

🧠 La logique & les commandes

Le truc : deux chronomètres millis indépendants (un par LED), chacun avec sa demi-période.

  • 1 Hz → période 1000 ms → bascule toutes les 500 ms.
  • 0,8 Hz → période 1/0,8 = 1250 ms → bascule toutes les 625 ms.

(La demi-période = le temps entre deux basculements, car un cycle complet = allumé + éteint.)

💻 Le code commenté 
#define LED1 8
#define LED2 10
unsigned long t1 = 0, t2 = 0;
const unsigned long DEMI1 = 500;   // 1 Hz   → demi-période 500 ms
const unsigned long DEMI2 = 625;   // 0,8 Hz → demi-période 625 ms
bool e1 = LOW, e2 = LOW;

void setup() {
  pinMode(LED1, OUTPUT);
  pinMode(LED2, OUTPUT);
}

void loop() {
  if (millis() - t1 >= DEMI1) { t1 = millis(); e1 = !e1; digitalWrite(LED1, e1); }
  if (millis() - t2 >= DEMI2) { t2 = millis(); e2 = !e2; digitalWrite(LED2, e2); }
}
✅ À tester / pièges
  • Les 2 LED se décalent peu à peu (fréquences différentes) → normal.
  • Piège : utiliser une seule variable de temps pour les deux → elles seraient synchronisées.

4 · ★ Le télérupteur

Manip 6 · cœur de l'examen
Cahier des chargesChaque appui sur le bouton change l'état de la LED (allumée ↔ éteinte) et elle y reste jusqu'au prochain appui. Une seule bascule par appui, et sans delay().
🔌 Le câblage
🧠 La logique & les commandes

Trois difficultés à gérer :

  1. Détecter un FRONT, pas un niveau : on ne bascule que lors du passage relâché→appuyé (sinon, tant qu'on tient, ça bascule en boucle).
  2. L'anti-rebond : on n'accepte le changement de niveau que s'il est stable depuis ~30 ms (le bouton « grésille » à la fermeture).
  3. Pas de delay() → on mesure le temps avec millis().

Méthode : on suit le niveau brut ; dès qu'il bouge, on (re)lance un chrono. Si le niveau est resté stable assez longtemps et qu'il est différent du dernier niveau confirmé, c'est un vrai changement → si c'est un appui (LOW), on bascule la LED.

💻 Le code commenté 
#define LED 8
#define BP  2
const unsigned long ANTIREBOND = 30;   // ms

bool etatLED = LOW;
bool dernierNiveau = HIGH;   // PULLUP : relâché = HIGH
bool niveauStable = HIGH;
unsigned long tChangement = 0;

void setup() {
  pinMode(LED, OUTPUT);
  pinMode(BP, INPUT_PULLUP);
  Serial.begin(9600);
}

void loop() {
  bool niveau = digitalRead(BP);

  // le niveau brut a bougé → on (re)lance le chrono anti-rebond
  if (niveau != dernierNiveau) {
    tChangement = millis();
    dernierNiveau = niveau;
  }

  // stable depuis assez longtemps ET réellement différent du dernier état confirmé
  if (millis() - tChangement > ANTIREBOND && niveau != niveauStable) {
    niveauStable = niveau;
    if (niveauStable == LOW) {     // front d'appui confirmé
      etatLED = !etatLED;          // ← on bascule UNE fois
      digitalWrite(LED, etatLED);
    }
  }
}
✅ À tester / pièges
  • Un appui = un seul changement, même si tu gardes le doigt → c'est le but.
  • Si la LED « saute » parfois 2 états : augmente ANTIREBOND (40-50 ms).
  • Piège classique : tester le niveau (if (digitalRead(BP)==LOW) etat=!etat;) → ça bascule des centaines de fois par appui. Il faut le front.

5 · Piloter une LED avec un potentiomètre

Manip 7
Cahier des chargesRégler la luminosité d'une LED avec un potentiomètre : du min au max, la luminosité varie sans interruption.
🔌 Le câblage
🧠 La logique & les commandes

On lit l'analogique (0 → 1023) et on le convertit en valeur PWM (0 → 255) avec map, qu'on envoie sur la LED avec analogWrite. La broche doit être marquée ~ (PWM).

💻 Le code commenté 
#define LED 9        // broche PWM (~)

void setup() {
  pinMode(LED, OUTPUT);
}

void loop() {
  int valeur = analogRead(A0);              // 0 → 1023
  int pwm = map(valeur, 0, 1023, 0, 255);   // converti en 0 → 255
  analogWrite(LED, pwm);                     // luminosité
}
✅ À tester / pièges
  • Tourne le potentiomètre : la LED varie en douceur.
  • Piège : choisir une broche non PWM (sans ~) → analogWrite ne fera qu'allumer/éteindre.
  • analogRead va jusqu'à 1023, analogWrite jusqu'à 255 : ne pas confondre les deux échelles.

6 · Le dimmer (appui maintenu)

Manip 8 · dépassement
Cahier des chargesUn appui prolongé sur le bouton fait varier la luminosité de la LED (PWM), et le sens s'inverse à chaque nouvel appui. La valeur s'arrête au max (255) ou au min (0).
🔌 Le câblage

D11 (~) → R 220 Ω → LED → GND et bouton entre D3 et GND (INPUT_PULLUP). La LED doit être sur une broche PWM (~).

🧠 La logique & les commandes

Tant que le bouton est maintenu, on fait évoluer une variable luminosite par petits pas, à cadence régulière (patron millis). On la borne entre 0 et 255 (constrain). Au relâchement (front montant), on inverse le sens — une seule fois.

💻 Le code commenté 
#define LED 11       // broche PWM
#define BP  3
int luminosite = 0;
bool sensMontant = true;
bool dernierAppuye = false;
unsigned long tPrecedent = 0;
const unsigned long PAS = 20;   // ms entre deux pas

void setup() {
  pinMode(LED, OUTPUT);
  pinMode(BP, INPUT_PULLUP);
}

void loop() {
  bool appuye = (digitalRead(BP) == LOW);

  if (appuye) {                              // tant qu'on maintient
    if (millis() - tPrecedent >= PAS) {
      tPrecedent = millis();
      luminosite += (sensMontant ? 5 : -5);  // monte ou descend
      luminosite = constrain(luminosite, 0, 255);  // borne 0..255
      analogWrite(LED, luminosite);
    }
  }

  if (dernierAppuye && !appuye) {   // front de relâchement
    sensMontant = !sensMontant;     // inverse le sens pour la prochaine fois
  }
  dernierAppuye = appuye;
}
✅ À tester / pièges
  • Maintiens : ça monte ; relâche, re-maintiens : ça descend.
  • Piège (présent dans le code original du labo) : inverser le sens dans un else sans front → il s'inverse des milliers de fois/seconde et le sens devient aléatoire. Il faut le front de relâchement.

7 · Compter les secondes d'appui

Comm · ex. 1
Cahier des chargesLors de l'appui sur un bouton, compter et afficher (moniteur série) le nombre de secondes écoulées tant que le bouton est enfoncé.
🔌 Le câblage

Bouton entre D5 et GND (INPUT_PULLUP). Ouvre le moniteur série (9600 bauds).

🧠 La logique & les commandes

Au front d'appui, on note l'heure de départ. Ensuite, tant qu'on appuie, on affiche un compteur qui augmente toutes les secondes (encore le patron millis, avec 1000 ms). C'est la base de toute mesure de durée → réutilisée pour l'appui long de l'examen.

💻 Le code commenté 
#define BP 5
unsigned long tDebut = 0;
unsigned long tDernierAffichage = 0;
bool dernierAppuye = false;

void setup() {
  pinMode(BP, INPUT_PULLUP);
  Serial.begin(9600);
}

void loop() {
  bool appuye = (digitalRead(BP) == LOW);

  if (appuye && !dernierAppuye) {     // front d'appui → départ du chrono
    tDebut = millis();
    tDernierAffichage = millis();
  }

  if (appuye) {                       // tant qu'on appuie, 1 affichage/seconde
    if (millis() - tDernierAffichage >= 1000) {
      tDernierAffichage += 1000;
      unsigned long secondes = (millis() - tDebut) / 1000;
      Serial.println(secondes);
    }
  }

  dernierAppuye = appuye;
}
✅ À tester / pièges
  • Maintiens : le moniteur affiche 1, 2, 3… une fois par seconde.
  • Piège (code original) : faire (tempsAppuye - tempsRelache)/1000 en continu donne n'importe quoi. Il faut mémoriser l'instant de départ au front.

8 · Afficher les lettres a → z

Comm · ex. 2
Cahier des chargesÀ l'appui sur le bouton, afficher les 26 lettres de l'alphabet (200 ms entre chaque). Si le bouton est toujours tenu après le z, ça recommence. Indice : type char + code ASCII.
🔌 Le câblage

Bouton entre D5 et GND (INPUT_PULLUP) + moniteur série.

🧠 La logique & les commandes

Un char est en réalité un nombre (code ASCII) : 'a' vaut 97, 'z' vaut 122. On peut donc faire lettre++ pour passer à la suivante. Tant que le bouton est tenu, on affiche une lettre toutes les 200 ms (patron millis), et après 'z' on revient à 'a'.

💻 Le code commenté 
#define BP 5
char lettre = 'a';                 // 'a' = 97 en ASCII
unsigned long tPrecedent = 0;
const unsigned long DELAI = 200;

void setup() {
  pinMode(BP, INPUT_PULLUP);
  Serial.begin(9600);
}

void loop() {
  if (digitalRead(BP) == LOW) {              // bouton maintenu
    if (millis() - tPrecedent >= DELAI) {
      tPrecedent = millis();
      Serial.println(lettre);
      lettre++;                              // lettre suivante
      if (lettre > 'z') { lettre = 'a'; }    // après z → retour à a
    }
  }
}
✅ À tester / pièges
  • Maintiens : a, b, c… z, puis ça reboucle.
  • Piège : déclarer lettre en int afficherait 97, 98… au lieu des lettres. Le type char est essentiel pour l'affichage en lettre.

9 · Afficher la luminosité d'une photorésistance

Comm · ex. 3
Cahier des chargesAfficher en continu la luminosité perçue par une photorésistance (lecture analogique), en %.
🔌 Le câblage
🧠 La logique & les commandes

La photorésistance + R forment un pont diviseur : la tension sur A0 change avec la lumière. On lit analogRead(A0) et on convertit en % avec map. Affichage cadencé (patron millis) pour ne pas inonder le moniteur.

💻 Le code commenté 
unsigned long tPrecedent = 0;
const unsigned long INTERVALLE = 500;

void setup() {
  Serial.begin(9600);
}

void loop() {
  if (millis() - tPrecedent >= INTERVALLE) {
    tPrecedent = millis();
    int valeur = analogRead(A0);              // 0 → 1023
    int pourcent = map(valeur, 0, 1023, 0, 100);
    Serial.print(pourcent);
    Serial.println(" %");
  }
}

Selon le branchement (LDR en haut ou en bas du pont), plus de lumière peut donner une valeur plus petite. Si c'est inversé, échange les bornes : map(valeur, 1023, 0, 0, 100).

✅ À tester / pièges
  • Couvre la photorésistance avec la main : le % doit bouger nettement.
  • Si le % bouge « à l'envers », inverse le map (voir ci-dessus).

10 · Allumer une LED par « ON » / « OFF »

Comm · ex. 4
Cahier des chargesAllumer une LED quand on envoie « ON » dans le moniteur série, l'éteindre avec « OFF ».
🔌 Le câblage

D8 → R 220 Ω → LED → GND. Moniteur série ouvert (9600), avec le champ de saisie en haut.

🧠 La logique & les commandes

On reçoit du texte : Serial.available() dit s'il y a quelque chose, Serial.readString() lit tout le message. .trim() enlève les espaces / retours à la ligne parasites. On compare ensuite la chaîne reçue à "ON" / "OFF".

💻 Le code commenté 
#define LED 8

void setup() {
  Serial.begin(9600);
  pinMode(LED, OUTPUT);
}

void loop() {
  if (Serial.available() > 0) {          // un message est arrivé ?
    String msg = Serial.readString();    // on lit tout
    msg.trim();                          // enlève espaces / \n
    if (msg == "ON")  { digitalWrite(LED, HIGH); }
    if (msg == "OFF") { digitalWrite(LED, LOW); }
  }
}
✅ À tester / pièges
  • Tape ON puis Entrée → LED allumée ; OFF → éteinte.
  • Piège : sans .trim(), le message peut contenir un retour à la ligne et ne jamais être égal à "ON".
  • La comparaison est sensible à la casse : on ≠ ON.

11 · Photorésistance sur requête « Lumi »

Comm · ex. 5
Cahier des chargesComme l'exercice 9, mais plus en continu : la luminosité n'est affichée que lorsqu'on envoie le message « Lumi » dans le moniteur.
🔌 Le câblage

Même pont diviseur que l'exercice 9 (5 V → LDR → A0 → R → GND) + moniteur série.

🧠 La logique & les commandes

On combine l'exercice 9 (lecture analogique) et l'exercice 10 (réception série) : seulement quand on reçoit « Lumi », on lit A0 et on affiche le %. C'est un système question → réponse.

💻 Le code commenté 
void setup() {
  Serial.begin(9600);
}

void loop() {
  if (Serial.available() > 0) {
    String msg = Serial.readString();
    msg.trim();
    if (msg == "Lumi") {                        // requête reçue
      int valeur = analogRead(A0);
      int pourcent = map(valeur, 0, 1023, 0, 100);
      Serial.print(pourcent);
      Serial.println(" %");
    }
  }
}
✅ À tester / pièges
  • Rien ne s'affiche tant que tu n'envoies pas Lumi → une mesure par requête.
  • Même piège du .trim() et de la casse qu'à l'exercice 10.

🎯 Capstone — l'examen complet

Examen type · /20
Énoncé (assemblage de tout) 1. Circuit LED + résistance sur le 5 V Arduino (choisir R pour ne pas griller la LED), mesurer U aux bornes de R au multimètre.
2. Programme télérupteur : chaque appui change l'état de la LED.
3. Ajouter une 2e LED (autre couleur) : un appui long (> 2 s) fait clignoter les 2 LED à 0,6 Hz.
4. Afficher dans la console le type d'appui (bref / long) et la période de clignotement mesurée (en s).
🔌 Le câblage

LED1 → D8 (+ R 220 Ω) → GND · LED2 (autre couleur) → D10 (+ R 220 Ω) → GND · bouton entre D2 et GND (INPUT_PULLUP).

Question 1 (calcul R) : R = (5 − VLED) / I = (5 − 2) / 0,02 = 150 Ω minimum (220 Ω = valeur sûre). Au multimètre, U aux bornes de R ≈ 5 − 2 = 3 V.

🧠 La logique & les commandes

On assemble 3 briques déjà vues :

  • Télérupteur anti-rebond (exercice 4) pour détecter les fronts proprement.
  • Mesure de durée (exercice 7) : entre le front d'appui et le relâchement → si < 2 s = bref, sinon = long.
  • Clignotement millis multi-LED (exercices 1-3) : 0,6 Hz → période 1/0,6 ≈ 1667 ms → bascule toutes les 833 ms.

Choix de comportement (à adapter selon la consigne du jour) : appui bref → bascule LED1 (télérupteur) et sort du mode clignotant ; appui long → les 2 LED clignotent. La période réelle est mesurée entre deux basculements (× 2 = période complète) et affichée.

💻 Le code commenté 
#define LED1 8
#define LED2 10
#define BP   2

const unsigned long ANTIREBOND   = 30;     // ms
const unsigned long SEUIL_LONG   = 2000;   // 2 s
const unsigned long DEMI_PERIODE = 833;    // 0,6 Hz → T≈1667 ms → demi 833 ms

bool etatLED1 = LOW;
bool modeClignotant = false;

// --- anti-rebond + fronts ---
bool dernierNiveau = HIGH, niveauStable = HIGH;
unsigned long tChangement = 0;
unsigned long tAppui = 0;        // instant du front d'appui

// --- clignotement + mesure de période ---
bool etatClignote = LOW;
unsigned long tClignote = 0;
unsigned long tDernierFront = 0;

void setup() {
  pinMode(LED1, OUTPUT);
  pinMode(LED2, OUTPUT);
  pinMode(BP, INPUT_PULLUP);
  Serial.begin(9600);
}

void loop() {
  // ===== anti-rebond + détection des fronts =====
  bool niveau = digitalRead(BP);
  if (niveau != dernierNiveau) { tChangement = millis(); dernierNiveau = niveau; }

  if (millis() - tChangement > ANTIREBOND && niveau != niveauStable) {
    niveauStable = niveau;
    if (niveauStable == LOW) {              // --- front d'appui ---
      tAppui = millis();
    } else {                               // --- front de relâchement → on classe ---
      unsigned long duree = millis() - tAppui;
      if (duree >= SEUIL_LONG) {
        Serial.println("Appui long");
        modeClignotant = true;             // les 2 LED vont clignoter
        tClignote = millis();
        tDernierFront = millis();
      } else {
        Serial.println("Appui bref");
        modeClignotant = false;            // retour en mode telerupteur
        etatLED1 = !etatLED1;              // bascule LED1
        digitalWrite(LED1, etatLED1);
        digitalWrite(LED2, LOW);
      }
    }
  }

  // ===== mode clignotant : 2 LED a 0,6 Hz =====
  if (modeClignotant) {
    if (millis() - tClignote >= DEMI_PERIODE) {
      tClignote = millis();
      etatClignote = !etatClignote;
      digitalWrite(LED1, etatClignote);
      digitalWrite(LED2, etatClignote);

      // periode reelle = 2 x (temps entre deux basculements)
      unsigned long periode = (millis() - tDernierFront) * 2;
      tDernierFront = millis();
      Serial.print("Periode mesuree : ");
      Serial.print(periode / 1000.0);
      Serial.println(" s");
    }
  }
}
✅ À tester / pièges
  • Appui bref → LED1 bascule, le moniteur affiche « Appui bref ».
  • Appui > 2 s → les 2 LED clignotent, le moniteur affiche « Appui long » puis « Periode mesuree : 1.67 s » (≈ 0,6 Hz). ✓
  • Adapte le comportement si la consigne diffère (ex. le clignotement démarre pendant l'appui long, ou un 2e appui arrête le clignotement).
