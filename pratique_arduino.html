🎯 Pratique Arduino

Montage sur Arduino réel : LED + bouton (télérupteur, appui long, console). Document autonome. La théorie/manips sont dans la synthèse labo.

Format & règles

⚠️ Conditions (lis-les !)
  • 2 h (évaluation comprise). Tu seras interrogé à l'oral sur ton programme.
  • Document autorisé (celui-ci !). Pas d'Internet, pas de Moodle, pas de GSM/montre connectée.
  • Si aucun programme ne fonctionne à la fin → note max = 4/20.
  • −6 points (cumulatifs) en cas de dégradation de matériel (LED grillée, multimètre…) ou de sabotage.
  • Triche → 0 + convocation.
💡 Stratégie Fais les questions dans l'ordre et teste après chaque étape : Q1 (circuit) → Q2 (télérupteur) → Q3 (appui long + clignotement) → Q4 (console). Un programme partiel qui marche vaut mieux qu'un programme complet qui plante. Câble propre, vérifie 2× avant d'alimenter.

Les 4 questions (énoncé type)

QÉnoncéPts
1Circuit LED + résistance sur l'alim 5 V de l'Arduino. Choisir la bonne résistance (ne pas griller la LED). Mesurer la tension aux bornes de la résistance au multimètre./3
2BP + LED : à chaque appui simple, la LED change d'état (télérupteur) et reste ainsi jusqu'au prochain appui./5
3Ajouter une 2e LED d'une autre couleur. Appui long (> 2 s) → les 2 LEDs clignotent à 0,6 Hz./7
4Console : afficher le type d'appui (bref/long) et la période de clignotement mesurée (en secondes)./5

Q1 · Circuit LED + résistance (/3)

Une LED a $V_{LED}\approx 2$ V et un courant nominal $I=20$ mA. Jamais de LED sans résistance. Sur 5 V :

📐 Résistance de limitation $$R=\frac{V_{alim}-V_{LED}}{I}=\frac{5-2}{0{,}02}=150\ \Omega$$ En pratique on prend 220 Ω ou 330 Ω (un peu plus = LED un poil moins lumineuse, mais sûr).
Mesure attendue Tension aux bornes de la résistance : $U_R=V_{alim}-V_{LED}=5-2=$ ≈ 3 V (à mesurer au multimètre, en parallèle sur la résistance). Avec 220/330 Ω la LED tire un peu moins de 20 mA, $U_R$ sera proche de 3 V.

Câblage Q1 : 5V → résistance → anode LED (patte longue) → cathode (méplat) → GND.

Câblage final (Q2 → Q4)

D2  ── BP ── GND                 (bouton : INPUT_PULLUP, pas de résistance externe)
D12 ── R(220Ω) ── LED1 ── GND    (LED couleur A)
D13 ── R(220Ω) ── LED2 ── GND    (LED couleur B)
💡 Rappels câblage LED : anode = patte longue (côté résistance/+), cathode = patte courte/méplat (côté GND). Bouton en INPUT_PULLUP : entre la broche et GND, appuyé = LOW. Évite les broches 0 et 1 (liaison série).

Programme complet (Q2 + Q3 + Q4) ⭐

💡 Anti-rebond robuste (la clé pour que ça marche en vrai) On ne lit pas le bouton « brutalement » : on confirme un changement seulement s'il reste stable plus de ~30 ms. Le rebond mécanique (faux fronts) est ignoré → le télérupteur ne bascule qu'une seule fois par appui. C'est la méthode officielle Arduino (Debounce) — bien plus fiable que de tester millis() au moment du front.
// ============================================================
//  Projet Arduino - LED + bouton
//  Q2 : telerupteur (un appui bref bascule LED1)
//  Q3 : appui long (>= 2 s) -> les 2 LEDs clignotent a 0,6 Hz
//  Q4 : console -> type d'appui + periode de clignotement mesuree
// ============================================================

#define BP    2     // bouton entre D2 et GND  (INPUT_PULLUP)
#define LED1  12    // LED couleur A (telerupteur)
#define LED2  13    // LED couleur B (clignotement)

// ----- Parametres -----
const unsigned long DEBOUNCE     = 30;    // anti-rebond (ms)
const unsigned long SEUIL_LONG   = 2000;  // appui >= 2 s = appui long
const unsigned long DEMI_PERIODE = 833;   // 0,6 Hz : T = 1666 ms -> T/2 = 833 ms

// ----- Anti-rebond robuste (methode officielle Arduino) -----
bool lectureBrute;                 // valeur lue maintenant
bool derniereLecture = HIGH;       // valeur lue au tour precedent
bool etatStable      = HIGH;       // etat CONFIRME (HIGH = relache, car PULLUP)
unsigned long tChangement = 0;     // instant du dernier changement de la lecture brute

// ----- Logique appui / LEDs -----
unsigned long tDebutAppui = 0;     // debut d'appui (front descendant confirme)
bool etatLed1 = LOW, etatLed2 = LOW;
bool clignote = false;             // mode clignotement actif ?
unsigned long tDerniereBascule = 0;

void setup() {
  pinMode(BP, INPUT_PULLUP);
  pinMode(LED1, OUTPUT);
  pinMode(LED2, OUTPUT);
  Serial.begin(9600);
}

void loop() {
  // ---------- 1) Lecture du bouton avec anti-rebond robuste ----------
  lectureBrute = digitalRead(BP);
  if (lectureBrute != derniereLecture) {
    tChangement = millis();          // la lecture vient de bouger -> on relance le chrono
  }
  // On CONFIRME le nouvel etat seulement s'il est stable depuis > DEBOUNCE ms
  if (millis() - tChangement > DEBOUNCE && lectureBrute != etatStable) {
    etatStable = lectureBrute;       // changement confirme = un VRAI front

    if (etatStable == LOW) {
      tDebutAppui = millis();         // front descendant : debut de l'appui
    } else {
      unsigned long duree = millis() - tDebutAppui;   // front montant : relachement

      if (duree >= SEUIL_LONG) {       // ----- APPUI LONG -----
        clignote = true;
        tDerniereBascule = millis();
        Serial.print("Appui long ("); Serial.print(duree); Serial.println(" ms)");
      } else {                          // ----- APPUI BREF : telerupteur LED1 -----
        clignote = false;
        etatLed2 = LOW; digitalWrite(LED2, LOW);
        etatLed1 = !etatLed1; digitalWrite(LED1, etatLed1);
        Serial.print("Appui bref ("); Serial.print(duree); Serial.println(" ms)");
      }
    }
  }
  derniereLecture = lectureBrute;

  // ---------- 2) Clignotement non bloquant des 2 LEDs ----------
  if (clignote && millis() - tDerniereBascule >= DEMI_PERIODE) {
    unsigned long periode = (millis() - tDerniereBascule) * 2;  // 2 demi-periodes = 1 cycle
    tDerniereBascule = millis();
    etatLed1 = !etatLed1; etatLed2 = !etatLed2;
    digitalWrite(LED1, etatLed1);
    digitalWrite(LED2, etatLed2);
    Serial.print("Periode mesuree : "); Serial.print(periode / 1000.0, 3); Serial.println(" s");
  }
}
Le calcul du 0,6 Hz (à savoir refaire) $T=\dfrac{1}{f}=\dfrac{1}{0{,}6}\approx 1666$ ms → demi-période $=\dfrac{T}{2}\approx 833$ ms (on bascule les LEDs toutes les 833 ms pour un rapport cyclique de 50 %).

Points clés à expliquer à l'oral

  • INPUT_PULLUP : bouton entre broche et GND, appuyé = LOW (logique inversée), évite une résistance externe.
  • Anti-rebond par état stable : on confirme après 30 ms → pas de double déclenchement (le bug classique du matériel réel).
  • Détection de front (sur l'état confirmé) : action une seule fois par appui (sinon ça se répète tant qu'on maintient).
  • millis() partout, jamais delay() : sinon impossible de clignoter ET lire le bouton en même temps.
  • 0,6 Hz → 833 ms de demi-période ; période mesurée = intervalle entre 2 bascules × 2.
  • unsigned long pour les temps (millis() déborde un int après ~32 s).

✅ Checklist avant de rendre

  • Une résistance sur chaque LED (jamais de LED nue)
  • Le programme compile sans erreur
  • Aucun delay() (recherche dans le code)
  • Télérupteur OK : 1 appui = 1 changement (pas 2-3)
  • Appui long (> 2 s) → les 2 LEDs clignotent à 0,6 Hz
  • Console : type d'appui (bref/long) + période (~1,666 s) affichés
  • Moniteur série réglé à 9600 bauds
  • Câblage propre, pas de fil qui dépasse
