0Setup & règles de sécurité

Checklist matériel

  • Carte Arduino UNO + câble USB
  • Breadboard + câbles mâle-mâle
  • LEDs de 2 couleurs différentes
  • Résistances : 150 Ω, 220 Ω, 330 Ω, 470 Ω, 10 kΩ
  • Bouton poussoir (BP), multimètre, PC avec Arduino IDE
⚠️ Ne pas perdre de points bêtement
  1. Jamais de LED sans résistance → claquage (−6 pts pour dégradation de matériel, cumulatif !).
  2. Sens LED : anode = patte longue (côté +), cathode = patte courte / méplat (côté −/GND).
  3. Ampèremètre en série (ouvrir le circuit), voltmètre en parallèle. Ampèremètre en parallèle = court-circuit.
  4. Brancher alimentation éteinte / Arduino débranché.
  5. Pull-up / pull-down obligatoire sur un BP, sinon entrée flottante.
💡 Méthodologie Lire l'énoncé en entier → dessiner le schéma → calculer la résistance LED → câbler et vérifier 2× → tester le hardware → coder par étapes → vérifier au moniteur série.

4Pont diviseur de tension

📐 $$U_{R2}=U_t\,\frac{R_2}{R_1+R_2} \qquad\text{(chargé)}\quad U_{R2}=U_t\,\frac{R_2\parallel R_{ch}}{R_1+(R_2\parallel R_{ch})}$$
⚠️ Le pont n'est valide qu'à vide ou avec une charge très grande (l'entrée A0 de l'Arduino, ≈ ∞, convient).

5Alimentation complète (labo 4)

Chaîne : secteur → transformateur → pont redresseur → condensateur → régulateur → sortie.

📐 Étapes de calcul 1) $V_{sec}=\frac{N_2}{N_1}V_{prim}$ · 2) $V_{crête}=V_{eff}\sqrt2$ · 3) pont : $V_{c,out}=V_{c,in}-1{,}4$, $V_{moy}=0{,}637V_{c,out}$, $f_{out}=2f$ · 4) $V_{ond}=\frac{V_{c,out}}{fRC}$, $r=\frac{V_{ond}}{V_{moy}}$ · 5) régulateur 78xx / LM317 $\left(V_{out}=1{,}25(1+\frac{R_2}{R_1})\right)$.
💡 Manip (ordre) 1) $V_2$ efficace au multimètre AC. 2) Période/fréquence à l'oscillo (50 Hz, 20 ms). 3) Après redresseur seul : $V_{moy}$ (DC) + $V_{crête}$ (oscillo). 4) Avec C=1 µF puis 100 µF : mesurer $V_{moy}$ et $V_{ond}$ → calculer $r$ et comparer. 5) Changer $R_{charge}$ → observer l'ondulation.
⚠️ Plus $C$ grand → ondulation petite. Plus $R_{charge}$ petite → ondulation grande.

6Photorésistance (LDR) — labo 5

Résistance qui diminue quand la lumière augmente. Beaucoup de lumière → R faible ; obscurité → R élevée.

📐$$U_{A0}=5\,\frac{R_{LDR}}{R+R_{LDR}}$$ analogRead(A0) renvoie 0–1023 (0–5 V).

Cahier des charges type : « sous un certain seuil de luminosité, la LED s'allume » → entrée analogique (A0) + sortie digitale (LED). Voir pattern LDR.

7Transistor NPN en commutation — labo 5

Interrupteur commandé : bloqué ($I_B\approx0$) → charge éteinte ; saturé (faible $I_B$) → charge allumée.

💡 Les 2 manips du labo (matériel : R 470 Ω + 10 kΩ, LED, transistor NPN) Manip 1 — commande directe par bouton : la base est pilotée par un bouton poussoir (via la R de 10 kΩ), pas par du code. L'Arduino ne sert que d'alim 5 V. Bouton relâché → base sans courant → LED éteinte ; appuyé → transistor saturé → LED allumée.
Manip 2 — commande par signal numérique : même circuit, mais la base est pilotée par une sortie Arduino (ex. D9) au lieu du bouton → on commande la LED par programme (digitalWrite).
📐 $$I_C=\beta I_B \quad I_E=I_C+I_B \quad V_{BE}\approx0{,}7\,\text{V} \quad I_{Csat}=\frac{V_{CC}}{R_C} \quad I_{Bmin}=\frac{I_{Csat}}{\beta} \quad R_B=\frac{V_{BB}-0{,}7}{I_B}$$
💡 Exemple (β=100, LED $V_F=2$V) $I_{Csat}=\frac{5-2}{470}=6{,}4$ mA → $I_{Bmin}=64\,\mu$A. Avec $R_B=10$ kΩ : $I_B=\frac{5-0{,}7}{10000}=430\,\mu$A ≫ 64 µA → saturé ✅. Avec $R_B=100$ kΩ : $I_B=43\,\mu$A < 64 µA → pas saturé.

10Analogique & PWM

Entrée analogique (ADC 10 bits : 0–1023, 0–5 V)

int v = analogRead(A0);              // 0..1023
float u = v * (5.0 / 1023.0);    // en volts

Sortie PWM (8 bits : 0–255) — broches ~ 3,5,6,9,10,11

analogWrite(9, 128);   // duty 50%
int pwm = map(v, 0, 1023, 0, 255);  // adapter analogRead → PWM
⚠️ analogRead = 0–1023 (10 bits), analogWrite = 0–255 (8 bits). Relier avec map().

11Patterns de code prêts à l'emploi

11.1 Clignotement non-bloquant avec millis() ⭐⭐⭐

#define LED 13
unsigned long tPrec = 0;
const unsigned long INTERVALLE = 500;
bool etat = LOW;
void setup(){ pinMode(LED, OUTPUT); }
void loop(){
  unsigned long t = millis();
  if (t - tPrec >= INTERVALLE){
    tPrec = t; etat = !etat;
    digitalWrite(LED, etat);
  }
  // on peut faire AUTRE chose ici (lire un BP...)
}
⚠️ Toujours unsigned long pour millis() (un int déborde après ~32 s). Éviter delay() qui bloque tout.

11.2 Télérupteur (anti-rebond, sans delay) ⭐⭐⭐

#define LED 13
#define BP1 2
bool etatLED = LOW, bpPrec = HIGH;
unsigned long tDernier = 0;
const unsigned long REBOND = 50;
void setup(){ pinMode(LED,OUTPUT); pinMode(BP1,INPUT_PULLUP); }
void loop(){
  bool bp = digitalRead(BP1);
  if (bpPrec==HIGH && bp==LOW){            // front descendant = appui
    if (millis()-tDernier > REBOND){
      etatLED = !etatLED;
      digitalWrite(LED, etatLED);
      tDernier = millis();
    }
  }
  bpPrec = bp;
}
💡 Pourquoi ça marche On agit sur le front (relâché→appuyé), pas tant que le BP est maintenu. L'anti-rebond ignore les micro-impulsions parasites (~10–50 ms).

📟Recueil de codes Arduino (labo)

Tes codes du labo (testés en vrai par un camarade), corrigés des bugs évidents. Le titre = la tâche. Les ⚠️ = correction apportée, les ✓ = correct tel quel. Pour chaque code : 🔌 câblage + 💡 pourquoi ce choix / alternative. Idéal pour câbler → tester → comprendre.

🔑 3 réflexes boutons (vrais sur le matériel) ① INPUT_PULLUP : bouton entre la broche et GND (pas de résistance externe). Appuyé = LOW (0), relâché = HIGH (1) → logique « inversée » (d'où les ! dans le code).
② Niveau vs front : lire le niveau (digitalRead) = « tant que c'est appuyé » ; détecter un front (HIGH→LOW) = « une seule fois par appui ».
③ Anti-rebond : un bouton mécanique « rebondit » (plusieurs bascules en ~10–50 ms). Pour une action par appui, on ignore les changements pendant ~50 ms — sinon ça se déclenche 2-3 fois (c'est LE bug classique sur matériel réel).

1 · LED allumée tant que le bouton est appuyé (2 boutons / 2 LEDs)

🔌 Câblage : LED1→D10, LED2→D13 (chacune + résistance ~220 Ω vers GND) ; BP1→D3, BP2→D2 (vers GND). 💡 Pourquoi : lecture de niveau + INPUT_PULLUP ; le ! rétablit « appuyé = allumé ». Pas besoin d'anti-rebond ici (on suit l'état, pas un front).

⚠️ corrigé L'original faisait digitalWrite(LED, etat) → la LED s'allumait quand le bouton était relâché (INPUT_PULLUP : appuyé = LOW). On inverse avec !.
#define LED1 10
#define BP1  3
#define LED2 13
#define BP2  2
void setup() {
  pinMode(BP1, INPUT_PULLUP);
  pinMode(BP2, INPUT_PULLUP);
  pinMode(LED1, OUTPUT);
  pinMode(LED2, OUTPUT);
}
void loop() {
  digitalWrite(LED1, !digitalRead(BP1));  // appuyé (LOW) -> !LOW = HIGH = LED ON
  digitalWrite(LED2, !digitalRead(BP2));
}

2 · Clignoter une LED avec delay()

🔌 LED→D8 (+ résistance). 💡 delay() est simple mais bloque tout le programme pendant l'attente. Alternative non bloquante → code 12.

✓ correct (delay = bloquant). 15 ms dans l'original → clignotement quasi invisible, mets ~500 ms.

#define LED 8
void setup() { pinMode(LED, OUTPUT); }
void loop() {
  digitalWrite(LED, HIGH);
  delay(500);
  digitalWrite(LED, LOW);
  delay(500);
}

3 · Clignoter 2 LEDs en alternance avec millis()

🔌 LED1→D8, LED2→D10. 💡 millis() = horloge non bloquante ; LED2 = !LED1 les met en opposition. Avantage sur delay : on pourrait lire un bouton en même temps.

⚠️ corrigé L'original écrivait pinMode(ledPin, OUTPUT) → variable inexistante (ne compile pas). On configure bien les deux broches.
#define LED1 8
#define LED2 10
int ledState1 = LOW;
unsigned long previousMillis = 0;
const int interval = 500;
void setup() {
  pinMode(LED1, OUTPUT);
  pinMode(LED2, OUTPUT);
}
void loop() {
  if (millis() - previousMillis >= interval) {
    previousMillis = millis();
    ledState1 = !ledState1;
    digitalWrite(LED1, ledState1);
    digitalWrite(LED2, !ledState1);   // toujours en opposition
  }
}

4 · Une LED clignote (millis), l'autre s'allume quand on appuie

🔌 LED1→D8 (suit le bouton), LED2→D10 (clignote), BP→D2. 💡 Montre qu'avec millis() on fait deux choses à la fois (clignoter ET lire le bouton) — impossible avec delay().

⚠️ corrigé Logique du bouton inversée (INPUT_PULLUP) → ajout du ! pour allumer LED1 à l'appui.
#define LED1 8    // suit le bouton
#define LED2 10   // clignote
#define BP   2
bool ledState2 = HIGH;
unsigned long previousMillis = 0;
const int interval = 500;
void setup() {
  pinMode(LED1, OUTPUT);
  pinMode(LED2, OUTPUT);
  pinMode(BP, INPUT_PULLUP);
}
void loop() {
  if (millis() - previousMillis >= interval) {
    previousMillis = millis();
    ledState2 = !ledState2;
    digitalWrite(LED2, ledState2);
  }
  digitalWrite(LED1, !digitalRead(BP));  // appuyé -> LED ON
}

5 · Dimmer : maintenir le bouton fait varier la luminosité (PWM)

🔌 LED→D11 (broche PWM ~, + résistance), BP→D3. 💡 analogWrite = PWM 0–255. Maintenir = varier, relâcher = inverser le sens. Ici delay(20) est acceptable (on reste volontairement dans la boucle tant qu'on appuie).

✓ correct. Tant qu'on maintient le bouton, la luminosité monte (ou descend) ; au relâchement, le sens s'inverse.

#define LED 11   // broche PWM (~)
#define BP  3
bool increasing = true;
int brightness = 0;
void setup() {
  pinMode(BP, INPUT_PULLUP);
  pinMode(LED, OUTPUT);
}
void loop() {
  if (digitalRead(BP) == LOW) {
    while (digitalRead(BP) == LOW) {
      brightness += increasing ? 5 : -5;
      brightness = constrain(brightness, 0, 255);
      analogWrite(LED, brightness);
      delay(20);
    }
    increasing = !increasing;   // au relâchement : on inverse le sens
  }
}

6 · Afficher la durée d'appui sur le bouton (en secondes)

🔌 BP→D5. 💡 Détection de front : on mémorise l'instant du début (front descendant) et on calcule au relâchement (front montant). C'est LE schéma pour mesurer une durée → réutilisé pour l'appui long.

⚠️ corrigé L'original calculait (tempsAppuye − tempsRelache) en continu → incohérent. Version correcte : on mémorise le début d'appui et on calcule la durée au relâchement (détection de front).
#define BP 5
unsigned long debut = 0;
bool appuye = false;
void setup() {
  pinMode(BP, INPUT_PULLUP);
  Serial.begin(9600);
}
void loop() {
  bool etat = (digitalRead(BP) == LOW);   // appuyé ?
  if (etat && !appuye) {                  // front : début d'appui
    debut = millis();
    appuye = true;
  }
  if (!etat && appuye) {                  // front : relâchement
    unsigned long duree = (millis() - debut) / 1000;
    Serial.print("Duree : "); Serial.print(duree); Serial.println(" s");
    appuye = false;
  }
}

7 · Afficher les lettres a → z à chaque appui

🔌 BP→D5 ; ouvre le moniteur série. 💡 char = code ASCII (97 = 'a'). Le test millis()-dernierTemps >= 200 sert de cadence ET d'anti-rebond grossier.

✓ logique correcte (accolades remises en ordre). Une lettre toutes les 200 ms tant qu'on appuie, puis on boucle.

#define BP 5
char lettre = 'a';                 // code ASCII 97
unsigned long dernierTemps = 0;
void setup() {
  pinMode(BP, INPUT_PULLUP);
  Serial.begin(9600);
}
void loop() {
  if (digitalRead(BP) == LOW && millis() - dernierTemps >= 200) {
    Serial.println(lettre);
    lettre++;
    if (lettre > 'z') lettre = 'a';  // 122 -> 97
    dernierTemps = millis();
  }
}

8 · Convertir une valeur analogique (A0) en pourcentage

🔌 potentiomètre (ou capteur) sur A0. 💡 analogRead = 0–1023 (10 bits) ; map() met à l'échelle 0–100. Affichage cadencé (500 ms) pour ne pas saturer le moniteur.

✓ correct.

unsigned long dernierTemps = 0;
const int intervalle = 500;
void setup() { Serial.begin(9600); }
void loop() {
  int valeur = analogRead(A0);
  valeur = map(valeur, 0, 1023, 0, 100);
  if (millis() - dernierTemps >= intervalle) {
    Serial.print(valeur); Serial.println("%");
    dernierTemps = millis();
  }
}

9 · Piloter une LED par "ON" / "OFF" reçus en série

🔌 LED→D8. 💡 readString() + .trim() (enlève le retour-ligne parasite). Tape "ON"/"OFF" dans le moniteur. Alternative : Serial.read() pour 1 seul caractère.

#define LED 8
void setup() { Serial.begin(9600); pinMode(LED, OUTPUT); }

10 · Afficher la luminosité (LDR) sur demande "lumi"

🔌 LDR en pont diviseur sur A0. 💡 Ne répond que si on tape "lumi". map(1023,0,…) inverse l'échelle (plus de lumière → % plus haut). Adapte les bornes selon ton câblage (LDR en haut ou en bas du pont).

✓ correct. map(valeur, 1023, 0, 0, 100) inverse l'échelle (plus de lumière → % plus élevé).

void setup() { Serial.begin(9600); }
void loop() {
  int valeur = analogRead(A0);
  valeur = map(valeur, 1023, 0, 0, 100);
  if (Serial.available() > 0) {
    String msg = Serial.readString();
    msg.trim();
    if (msg == "lumi") { Serial.print(valeur); Serial.println("%"); }
  }
}

11 · Allumer une LED selon la luminosité (LDR + seuil)

🔌 LDR sur A0, LED→D7. 💡 Seuil à 50 % : sombre → LED ON. Conseil : affiche d'abord les valeurs au moniteur pour choisir le bon seuil selon ta pièce.

⚠️ corrigé Supprimer la ligne #define A0 : elle « écrasait » A0 (vide) → analogRead(A0) ne compilait plus. A0 existe déjà.
#define LED 7
void setup() { Serial.begin(9600); pinMode(LED, OUTPUT); }
void loop() {
  int valeur = analogRead(A0);
  valeur = map(valeur, 0, 1023, 0, 100);
  if (valeur < 50) digitalWrite(LED, HIGH);  // sombre -> LED ON
  else             digitalWrite(LED, LOW);
}

12 · Clignoter une LED avec millis() (non bloquant)

🔌 LED→D9. 💡 LA référence : pendant l'« attente », le programme reste libre (on pourrait lire un bouton). À préférer à delay() dès qu'il y a plusieurs tâches simultanées.

✓ correct — la référence du clignotement non bloquant.

#define LED 9
unsigned long tempsPrecedent = 0;
bool ledState = HIGH;
void setup() { pinMode(LED, OUTPUT); }
void loop() {
  if (millis() - tempsPrecedent > 500) {
    tempsPrecedent = millis();
    ledState = !ledState;
  }
  digitalWrite(LED, ledState);
}

Pièges classiques ⚠️

  1. LED sans résistance → claquage (−6 pts).
  2. int au lieu de unsigned long pour millis() → débordement après ~32 s.
  3. delay() interdit dans les cahiers des charges → utiliser millis().
  4. Confondre analogRead (0–1023) et analogWrite (0–255) → map().
  5. Pull-up oublié → BP imprévisible. INPUT_PULLUP : LOW = appuyé.
  6. Anti-rebond oublié → le télérupteur bascule 2-3 fois.
  7. Détection sur état au lieu de front → action répétée tant que maintenu.
  8. Pas de Serial.begin() ou mauvais baud rate → console illisible.
  9. Broches 0/1 utilisées → upload bloqué.

Avant de rendre

  • Le programme compile sans erreur
  • Aucun delay()
  • Télérupteur OK (1 appui = 1 changement)
  • Appui long déclenche le clignotement à 0,6 Hz
  • Console affiche type d'appui + période
  • Résistance présente sur chaque LED, câblage propre
